# Документация sixlmx

Документация предназначена для участников организации sixlmx: студентов, практикантов и наставников, которые работают с учебными проектами, заданиями и репозиториями.

Здесь собрана базовая информация о том, как начать работу в организации, где искать свой проект, как выполнять задания и как отправлять изменения на проверку.

## Что такое sixlmx?

sixlmx — рабочая GitHub-организация для учебной разработки и практики на реальных задачах.

В организации будут находиться репозитории с учебными проектами, заданиями, документацией и практическими работами. Одно из основных направлений — разработка образовательных веб-продуктов: приложения для расписания, публичной части для просмотра расписания и инструментов публикации.

## Как начать работу

1. Примите приглашение в организацию.
2. Убедитесь, что вошли в GitHub под нужным аккаунтом.
3. Найдите репозиторий со своим заданием или проектом.
4. Откройте README конкретного репозитория.
5. Изучите требования, команды запуска и правила сдачи.
6. Выполняйте работу в отдельной ветке, если это требуется заданием.
7. Отправляйте изменения на проверку через pull request.

## Как работать с учебным репозиторием

В каждом репозитории должна быть своя инструкция. Обычно в ней указываются:

- цель задания или проекта;
- как установить зависимости;
- как запустить проект;
- какие команды использовать для проверки;
- что считается готовым результатом;
- как отправить решение на ревью.

Если в README конкретного репозитория есть отдельные правила, сначала нужно следовать им.

## Как отправлять изменения на проверку

Перед отправкой изменений проверьте:

- проект запускается;
- нужные проверки выполнены;
- в commit не попали секреты, токены, пароли и лишние файлы;
- commit message понятный;
- pull request относится к одной задаче;
- описание PR коротко объясняет, что было сделано.

Если задание требует другой порядок сдачи, используйте порядок из README конкретного репозитория.

## Как сообщать о проблемах

Если что-то не получается, подготовьте информацию заранее:

- ссылку на репозиторий;
- что вы хотели сделать;
- какую команду запускали;
- какой результат ожидали;
- какой текст ошибки получили;
- скриншот, если проблема в интерфейсе.

Чем точнее описана проблема, тем быстрее её можно разобрать.

## Основные правила

- Не коммитить секреты, токены, пароли и личные данные.
- Не менять чужие ветки и чужие репозитории без разрешения.
- Не делать force push без согласования.
- Не смешивать несколько разных задач в одном pull request.
- Писать понятные commit messages.
- Проверять работу проекта перед сдачей.
- Если возникла ошибка, сначала сохранить команду, вывод терминала и контекст проблемы.
